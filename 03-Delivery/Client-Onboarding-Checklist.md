## Section 1: Pre-Kickoff (Internal)

## Section 2: Information Gathering (From Client)

### Business Information
- [ ] Organization chart / team structure (who does what)
- [ ] List of current software tools and logins
- [ ] Brand guidelines (logo, colors, tone of voice) — if relevant
- [ ] Sample documents / communications we'll be automating

### Technical Access
- [ ] CRM access (admin or integration-level)
- [ ] Email platform access
- [ ] Calendar system access
- [ ] Phone system / SMS platform access (if applicable)
- [ ] Website admin access (if applicable)
- [ ] Any other tool access needed: [list]

### Stakeholder Info
- [ ] Primary point of contact (name, email, phone)
- [ ] Technical point of contact (if different)
- [ ] Decision-maker for approvals
- [ ] End users who will interact with the automation (names + roles)

---

## Section 3: Kickoff Call

- [ ] Introductions completed
- [ ] Scope and timeline reviewed and confirmed
- [ ] Communication preferences confirmed (channel, frequency, response time)
- [ ] Access requirements reviewed — client knows what to send
- [ ] First check-in date scheduled
- [ ] Client questions answered
- [ ] Kickoff summary email sent within 24 hours

---

## Section 4: Build Phase Readiness

- [ ] All required access/credentials received and tested
- [ ] Sample data or content available for building
- [ ] Integration endpoints confirmed and accessible
- [ ] Any compliance or security requirements documented
- [ ] Build timeline shared with client
- [ ] Internal build checklist created per SOP

---

## Section 5: Pre-Launch

- [ ] All automations internally tested
- [ ] Client UAT session completed
- [ ] Client feedback incorporated
- [ ] User documentation delivered
- [ ] Training session scheduled
- [ ] Launch date confirmed with client

---

## Section 6: Post-Launch

- [ ] All automations live and running in production
- [ ] Training session completed and recorded
- [ ] "You're Live!" email sent with:
  - [ ] Documentation links
  - [ ] Support contact info
  - [ ] FAQ / troubleshooting guide
- [ ] First 48-hour monitoring active
- [ ] Week 1 check-in scheduled
- [ ] Support expectations confirmed

---

## Section 7: Handoff to Support

- [ ] Active monitoring transitioned to support cadence
- [ ] All credentials and access documented in project folder
- [ ] Automation architecture documented (what connects to what)
- [ ] Known issues or quirks documented
- [ ] Support tier confirmed (Starter / Growth / Premium)
- [ ] Expansion opportunities noted for Month 3 review
